Items are consumable objects, materials, and special goods that players can acquire, use, and trade. Unlike equipment (weapons, armor, units), items are typically consumed on use or used for crafting and enhancement.

## Item Rarity System

Items have rarity tiers that affect their availability and power:

### Rarity Tiers
1. **Common** (White): Easily obtained, frequently dropped
2. **Uncommon** (Green): Moderately rare, decent drops
3. **Rare** (Blue): Hard to find, powerful effects
4. **Very Rare** (Purple): Very hard to find, exceptional effects
5. **Legendary** (Orange/Gold): Extremely rare, unique items

## Item Usage

### Using Items

#### In Combat
- Open quick menu or inventory
- Select item
- Use on self or target ally (for support items)
- Item is consumed immediately

#### Out of Combat
- Access full inventory
- Use recovery items to restore HP/TP
- Feed items to MAG
- Cannot use quest items outside quest context

### Item Limits

- Inventory capacity: 30 slots (see [Inventory](/mechanics/inventory))
- Stack limits vary by item type
- Some items cannot be used in certain areas
- Quest items may have usage restrictions

## Database Schema

```sql
-- Item definitions
CREATE TABLE item_definitions (
  item_id VARCHAR(100) PRIMARY KEY,
  name VARCHAR(255) NOT NULL,
  description TEXT,
  item_category VARCHAR(50) NOT NULL, -- recovery, status_cure, buff, mag_food, material, currency, quest, trap, cosmetic
  item_type VARCHAR(50) NOT NULL, -- monomate, grinder, photon_drop, etc.
  rarity INTEGER NOT NULL,

  -- Stack properties
  stackable BOOLEAN DEFAULT FALSE,
  max_stack_size INTEGER DEFAULT 1,

  -- Usage
  consumable BOOLEAN DEFAULT TRUE,
  usable_in_combat BOOLEAN DEFAULT TRUE,
  target_type VARCHAR(20), -- self, ally, enemy, area, none

  -- Effects (JSONB for flexibility)
  effect_data JSONB, -- hp_restore: 500, tp_restore: 100, status_cure: "poison", etc.

  -- Trading
  tradeable BOOLEAN DEFAULT TRUE,
  sellable BOOLEAN DEFAULT TRUE,
  sell_price INTEGER,
  buy_price INTEGER,

  -- Visual
  icon_path TEXT NOT NULL,
  model_path TEXT, -- For items that appear in world

  INDEX idx_category (item_category),
  INDEX idx_rarity (rarity),
  INDEX idx_type (item_type)
);

-- Item effects (for complex multi-effect items)
CREATE TABLE item_effects (
  effect_id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  item_id VARCHAR(100) NOT NULL,
  effect_type VARCHAR(50) NOT NULL, -- restore_hp, restore_tp, cure_status, buff_stat, etc.
  effect_value JSONB NOT NULL, -- Flexible storage for effect parameters
  duration_seconds INTEGER, -- For timed effects

  FOREIGN KEY (item_id) REFERENCES item_definitions(item_id),
  INDEX idx_item_effects (item_id)
);

-- Player item instances (in inventory or storage)
-- See inventory_items table in Inventory documentation
```

## Server API

### Use Item

**Endpoint**: `POST /api/items/use`

**Request**:
```json
{
  "itemInstanceId": "uuid-v4",
  "targetCharacterId": "uuid-v4-or-null-for-self",
  "quantity": 1
}
```

**Response**:
```json
{
  "success": true,
  "itemId": "monomate",
  "effectsApplied": [
    {
      "type": "restore_hp",
      "amount": 500,
      "targetId": "character-uuid"
    }
  ],
  "remainingQuantity": 9
}
```

### Get Item Info

**Endpoint**: `GET /api/items/:itemId`

**Response**:
```json
{
  "itemId": "monomate",
  "name": "Monomate",
  "description": "Restores a small amount of HP",
  "category": "recovery",
  "rarity": 1,
  "stackable": true,
  "maxStackSize": 99,
  "sellPrice": 50,
  "buyPrice": 100,
  "effects": [
    {
      "type": "restore_hp",
      "value": 500
    }
  ]
}
```

## Item Acquisition

### Shops

Items can be purchased from NPCs:
- Medical shops: Recovery and cure items
- Tool shops: Materials and grinders
- Special traders: Rare items for Photon Drops

### Enemy Drops

Items drop from defeated enemies:
- Drop rates vary by enemy type and rarity
- Rare enemies have better drop tables
- Boss enemies drop rare and unique items

### Quest Rewards

Completing quests grants item rewards:
- Fixed rewards for story quests
- Random rewards from repeatable quests
- Bonus items for quest modifiers (time, no damage, etc.)

### Treasure Boxes

Items found in breakable boxes and containers:
- Random items based on area and difficulty
- Rare boxes contain better loot
- Some boxes are hidden or require puzzle solving
