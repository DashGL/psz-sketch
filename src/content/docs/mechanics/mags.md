MAGs are floating companion creatures that provide stat bonuses and special abilities to their owner. Every character receives a starter MAG at level 5 during character creation, and MAGs can be fed items to grow and evolve.

## MAG Basics

### Starting MAG

- All characters receive a MAG at creation
- Starting level: 5
- Player chooses the MAG's color during character creation
- Base stats: 0 POW, 0 DEX, 0 MIND

### MAG Stats

MAGs have three primary stats that grow through feeding:

#### POW (Power)
- Increases owner's ATP (Attack Power)
- Fed by weapons and power-focused items
- Influences physical combat performance

#### DEX (Dexterity)
- Increases owner's ATA (Attack Accuracy) and defensive stats
- Fed by defense items and certain consumables
- Influences accuracy and defense

#### MIND (Mental)
- Increases owner's MST (Mental Strength)
- Fed by technique-based items and monomates
- Influences technique power

### Additional MAG Properties

#### Synchro
- Represents the bond between character and MAG
- Affects photon blast power
- Range: 0-120%

#### IQ
- Affects MAG's behavior and autonomous actions
- Determines when MAG uses support abilities
- Influenced by feeding patterns

#### Level
- MAG level increases as stats grow
- Maximum level: TBD
- Level = (POW + DEX + MIND) / 100 + 5

## Feeding System

### How to Feed

Players can feed items to their MAG from inventory:
- Weapons increase POW
- Armor/Frames increase DEX
- Consumables (Monomates, etc.) increase MIND
- Each item provides different stat distributions

### Feeding Mechanics

- Each item provides a certain amount of stat growth
- Feeding too much of one stat type can affect evolution
- MAG's current stats determine which evolution path is available
- Overfeeding can temporarily reduce synchro

## MAG Evolution

MAGs evolve based on their stat distribution at certain level thresholds. There are four evolution forms (plus rare evolutions) that determine the MAG's appearance and photon blast.

### Evolution Triggers

MAGs evolve at specific level milestones:
- **Level 10**: First Form → Second Form
- **Level 35**: Second Form → Third Form
- **Level 60**: Third Form → Fourth Form
- **Level 60+**: Rare Forms (with special Heart items)

### First Form: Mag

**Appearance**: A round blob

**Photon Blast**: None

**Obtaining**:
- Starting MAG for new characters
- Purchase from Photon Drop Trader for 5 Photon Drops (random color)
- Create new character and transfer MAG via Player Trunk

## Photon Blasts

Photon Blasts are powerful special attacks unleashed by MAGs. Each MAG evolution has an associated photon blast that provides unique offensive or supportive effects.

For detailed information about each photon blast, see the [Photon Blasts](/mechanics/photon-blasts) article.

### Available Photon Blasts

- **[Guraniru](/mechanics/photon-blasts#guraniru)**: Associated with power-focused MAGs
- **[Midogaru](/mechanics/photon-blasts#midogaru)**: Associated with defense-focused MAGs
- **[Pashifaru](/mechanics/photon-blasts#pashifaru)**: Associated with accuracy-focused MAGs
- **[Furoojiru](/mechanics/photon-blasts#furoojiru)**: Associated with mind-focused MAGs

### Activation

- Requires sufficient synchro level
- Consumes synchro to activate
- Powerful area-of-effect or single-target attacks

### Synchro Building

- Synchro increases during combat
- Taking damage increases synchro
- Dealing damage increases synchro
- Synchro decreases when using photon blast

## MAG Actions

MAGs can perform autonomous actions to assist their owner:

### Support Actions

- **Healing**: Restores HP when owner is low
- **Resurrection**: Revives fallen owner (consumes synchro)
- **Invincibility**: Temporary invulnerability (high synchro required)

### Trigger Conditions

MAG IQ and synchro level determine when these actions activate.

## Database Schema

```sql
-- MAG definitions (base MAG types/forms)
CREATE TABLE mag_definitions (
  mag_type VARCHAR(50) PRIMARY KEY,
  name VARCHAR(255) NOT NULL,
  description TEXT,
  required_pow INTEGER,
  required_dex INTEGER,
  required_mind INTEGER,

  -- Stat bonuses per level
  pow_bonus_multiplier DECIMAL(5,2) DEFAULT 1.0,
  dex_bonus_multiplier DECIMAL(5,2) DEFAULT 1.0,
  mind_bonus_multiplier DECIMAL(5,2) DEFAULT 1.0,

  -- Photon blast
  photon_blast_id VARCHAR(50),

  -- Visual
  model_path TEXT NOT NULL,

  INDEX idx_mag_type (mag_type)
);

-- Character MAG state (extends table from character-create.md)
CREATE TABLE character_mag (
  mag_id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  character_id UUID NOT NULL,
  mag_type VARCHAR(50) NOT NULL DEFAULT 'starter', -- Current evolution form
  level INTEGER NOT NULL DEFAULT 5,
  color_index INTEGER NOT NULL,

  -- Stats
  pow INTEGER NOT NULL DEFAULT 0,
  dex INTEGER NOT NULL DEFAULT 0,
  mind INTEGER NOT NULL DEFAULT 0,
  synchro DECIMAL(5,2) NOT NULL DEFAULT 0,
  iq INTEGER NOT NULL DEFAULT 0,

  -- Photon blast
  photon_blast VARCHAR(50),

  -- Metadata
  created_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP,
  last_fed_at TIMESTAMP WITH TIME ZONE,

  FOREIGN KEY (character_id) REFERENCES character_state(character_id),
  FOREIGN KEY (mag_type) REFERENCES mag_definitions(mag_type)
);

-- MAG feeding history (for event sourcing)
CREATE TABLE mag_feeding_events (
  event_id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  mag_id UUID NOT NULL,
  item_id VARCHAR(100) NOT NULL,
  pow_gain INTEGER NOT NULL DEFAULT 0,
  dex_gain INTEGER NOT NULL DEFAULT 0,
  mind_gain INTEGER NOT NULL DEFAULT 0,
  iq_change INTEGER NOT NULL DEFAULT 0,
  fed_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP,

  FOREIGN KEY (mag_id) REFERENCES character_mag(mag_id),

  INDEX idx_mag_history (mag_id, fed_at)
);

-- MAG evolution history
CREATE TABLE mag_evolution_events (
  event_id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  mag_id UUID NOT NULL,
  from_mag_type VARCHAR(50),
  to_mag_type VARCHAR(50) NOT NULL,
  evolution_level INTEGER NOT NULL,
  evolved_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP,

  FOREIGN KEY (mag_id) REFERENCES character_mag(mag_id),

  INDEX idx_mag_evolutions (mag_id, evolved_at)
);

-- Photon blast definitions
CREATE TABLE photon_blast_definitions (
  blast_id VARCHAR(50) PRIMARY KEY,
  name VARCHAR(255) NOT NULL,
  description TEXT,
  damage_multiplier DECIMAL(5,2),
  area_of_effect BOOLEAN DEFAULT FALSE,
  synchro_cost DECIMAL(5,2) NOT NULL,
  animation_path TEXT
);
```

## Server API

### Feed MAG

**Endpoint**: `POST /api/mag/feed`

```json
{
  "magId": "uuid-v4",
  "itemInstanceId": "uuid-v4"
}
```

**Response**: Updated MAG stats and evolution status

### Use Photon Blast

**Endpoint**: `POST /api/mag/photon-blast`

```json
{
  "magId": "uuid-v4",
  "targetId": "enemy-uuid-or-null-for-area"
}
```
